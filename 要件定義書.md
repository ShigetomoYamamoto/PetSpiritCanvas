要件定義書

1. システム概要

1-1. システム名

仮称：ペット幻想合成画像メーカー

1-2. 目的

ユーザーがアップロードした風景画像とペット画像をもとに、
ペットを背景から切り抜き、風景上に半透明で重ねた幻想的な合成画像をブラウザ上で簡単に作成できるWebアプリを提供する。

また、作成した画像を
	•	端末へダウンロード
	•	SNSへ投稿しやすい形で共有

できるようにする。

1-3. 開発方針

本システムは、爆速開発・低コスト公開・運用負荷最小化を優先し、以下の方針で開発する。
	•	バックエンドなし、または極小
	•	画像処理は基本的にブラウザ内で完結
	•	ユーザー画像はサーバー保存しない
	•	MVPを最小構成で早期公開
	•	スマホ完全最適化よりも、まずはPC・モバイル主要環境で使えることを優先

⸻

2. スコープ

2-1. 対象範囲

本リリースで対象とするのは以下。
	•	風景画像アップロード
	•	ペット画像複数アップロード
	•	ペット画像の背景除去
	•	ペット画像の配置、拡大縮小、透明度調整
	•	レイヤー順制御
	•	合成画像のプレビュー
	•	合成画像の書き出し
	•	端末へのダウンロード
	•	SNS投稿導線の提供

2-2. 対象外

初期リリースでは以下を対象外とする。
	•	会員登録、ログイン
	•	作品のサーバー保存
	•	作品の下書き保存
	•	他ユーザーとの共有ギャラリー
	•	コメント機能
	•	画像編集の高機能化（色調補正、フィルタ、エフェクト多数）
	•	動画出力
	•	AIによる自動レイアウト提案
	•	ネイティブアプリ化

⸻

3. 想定ユーザー

3-1. 主対象ユーザー
	•	ペット写真を思い出として加工したい一般ユーザー
	•	SNSに投稿する画像を簡単に作りたいユーザー
	•	難しい画像編集ソフトなしで、雰囲気のある画像を作りたいユーザー

3-2. ユーザー環境
	•	PCブラウザ利用を主対象
	•	スマホブラウザでも利用可能とするが、性能制約に応じて制限あり
	•	対応ブラウザは Chrome / Safari / Edge の最新版を優先

⸻

4. システム方式

4-1. 全体方針
	•	フロントエンドのみで成立する構成を基本とする
	•	画像処理はクライアントサイドで行う
	•	静的サイトとしてホスティングする

4-2. 想定技術構成
	•	フロントエンド: Vite + Vue 3 + TypeScript
	•	UI描画: Konva.js または Canvas API
	•	背景除去: @imgly/background-removal
	•	AI推論: ONNX Runtime Web
	•	配信: Cloudflare Pages
	•	分析: 必要なら軽量なアクセス解析のみ導入

4-3. 採用理由
	•	Vue系は既存スキルに寄せやすい
	•	Viteで初速が速い
	•	バックエンド不要で開発・運用が軽い
	•	Cloudflare Pagesで無料公開しやすい
	•	バズ時のサーバー負荷リスクを下げやすい

⸻

5. 機能要件

5-1. 画像アップロード機能

概要

ユーザーは風景画像1枚と、ペット画像複数枚をアップロードできる。

要件
	•	風景画像は1枚アップロード可能
	•	ペット画像は複数枚アップロード可能
	•	初期リリースでは最大3枚までとする
	•	対応形式は PNG / JPEG / WebP
	•	非対応形式の場合はエラーメッセージを表示する
	•	アップロード画像はブラウザ内のみで処理し、サーバー送信しない

制限
	•	1ファイルあたりの推奨上限サイズを設ける
	•	極端に大きい画像は内部でリサイズして利用する

⸻

5-2. 背景除去機能

概要

アップロードしたペット画像の背景を除去し、切り抜き済み画像として合成に利用する。

要件
	•	ペット画像に対して背景除去を実行できる
	•	背景除去はブラウザ内で実行する
	•	画像ごとに順次処理する
	•	背景除去後の結果をプレビュー表示する
	•	処理中はローディング状態を表示する
	•	背景除去に失敗した場合、再実行または削除ができる

制約
	•	端末性能によって処理時間が変動する
	•	メモリ不足回避のため、入力画像は内部的に縮小する場合がある

⸻

5-3. 合成編集機能

概要

背景除去したペット画像を風景画像上に重ねて、自由に配置・調整できる。

要件
	•	ペット画像ごとに以下の編集ができる
	•	位置移動
	•	サイズ変更
	•	透明度変更
	•	回転
	•	表示 / 非表示切替
	•	削除
	•	ペット画像ごとにレイヤー順を変更できる
	•	前面へ
	•	背面へ
	•	キャンバス上でドラッグ操作できる
	•	選択中レイヤーに対する編集パネルを表示する

初期値
	•	ペット画像追加時は、中央付近に適切なサイズで仮配置する

⸻

5-4. プレビュー機能

概要

編集内容をリアルタイムにプレビューできる。

要件
	•	編集結果を即時反映する
	•	プレビュー表示は操作性優先の軽量解像度とする
	•	書き出し時は高解像度で再描画する

⸻

5-5. 書き出し機能

概要

作成した合成画像を1枚の画像として生成できる。

要件
	•	合成画像を PNG または JPEG で書き出せる
	•	初期リリースでは PNG固定 でも可
	•	書き出し前に最終レンダリングを行う
	•	書き出し中は処理状態を表示する

⸻

5-6. ダウンロード機能

概要

生成した画像をユーザー端末に保存できる。

要件
	•	書き出した画像を端末にダウンロードできる
	•	ファイル名は規則的に付与する
	•	例: pet-memory-YYYYMMDD-HHmmss.png
	•	ダウンロード後に完了メッセージを表示する

⸻

5-7. SNS共有機能

概要

生成した画像をSNSへ投稿しやすくする。

要件

初期リリースでは、確実に短期間で実現できる方式を採用する。

MVP方式
	•	「Xに投稿する」ボタンを表示する
	•	ボタン押下で、Xの投稿画面を新規タブで開く
	•	投稿文の初期文言をセットできる
	•	ハッシュタグの初期値をセットできる

注意事項
	•	ブラウザ制約上、画像ファイルそのものをWebからX投稿画面へ自動添付することは行わない
	•	そのため、ユーザー導線は以下とする
	1.	画像をダウンロード
	2.	X投稿画面を開く
	3.	ユーザーが手動で画像を添付して投稿

投稿文例
	•	うちの子の思い出画像を作りました。
	•	#ペット #思い出 #画像メーカー

将来拡張
	•	Web Share API 対応端末では共有シートを利用する
	•	SNS別の共有テンプレートを追加する

⸻

5-8. Web Share API対応機能（任意・対応環境のみ）

概要

モバイルや一部対応環境では、共有シートを開けるようにする。

要件
	•	対応ブラウザであれば共有ボタンを表示する
	•	非対応環境では非表示、または代替UIを表示する
	•	初期リリースでは必須ではないが、工数が軽ければ導入する

⸻

5-9. エラーハンドリング機能

要件

以下の場合に適切なメッセージを表示する。
	•	画像形式が非対応
	•	画像サイズが大きすぎる
	•	背景除去に失敗した
	•	メモリ不足で処理継続が困難
	•	書き出しに失敗した

⸻

6. 非機能要件

6-1. パフォーマンス
	•	ペット画像の同時登録数は初期リリースで最大3枚
	•	背景除去処理は並列ではなく逐次実行
	•	プレビューは軽量解像度で描画する
	•	書き出し時のみ高解像度化する
	•	スマホでは必要に応じて内部解像度を自動縮小する

6-2. 可用性
	•	静的ホスティングで公開する
	•	バックエンドを持たないため、サーバーダウン要因を最小化する

6-3. セキュリティ
	•	ユーザー画像はサーバー保存しない
	•	API送信しない
	•	個人情報を収集しない
	•	必要最小限の外部スクリプトのみ利用する

6-4. 保守性
	•	コンポーネント単位で分割する
	•	画像処理ロジックはUI層から分離する
	•	レイヤー管理を単一ストアで扱う

6-5. コスト
	•	無料または極小コスト運用を前提とする
	•	Cloudflare Pagesでの公開を前提とする

⸻

7-1. トップ画面

	•	ペット画像アップロード
	•	作業エリアへの導線

7-2. 編集画面

表示領域

	•	合成プレビューキャンバス
	•	右サイド
	•	選択中レイヤーの編集UI

右サイド編集項目
	•	透明度スライダー
	•	サイズ変更
	•	回転
	•	前面 / 背面
	•	削除

7-3. 完成操作エリア
	•	ダウンロードボタン
	•	X投稿ボタン
	•	共有ボタン（対応環境のみ）

⸻

8. 利用フロー

8-1. 基本フロー
	1.	ユーザーが風景画像をアップロードする
	2.	ユーザーがペット画像を1〜3枚アップロードする
	3.	システムがペット画像の背景除去を順次実行する
	4.	ユーザーがペット画像の位置・サイズ・透明度を調整する
	5.	ユーザーが合成結果を確認する
	6.	ユーザーが画像を書き出す
	7.	ユーザーが端末へダウンロードする
	8.	必要に応じてX投稿画面を開く

⸻

9. UX要件
	•	初見でも迷わないUIにする
	•	機能を絞り、ボタン数を増やしすぎない
	•	編集項目は必要最小限に限定する
	•	画像加工経験のない人でも3分以内に完成できることを目指す
	•	「ダウンロードして投稿」の導線を明確に表示する

⸻

10. 制約・前提条件
	•	背景除去精度は入力画像に依存する
	•	古い端末や低スペック端末では処理が重くなる場合がある
	•	SNSごとの仕様変更により共有導線は将来的に調整が必要
	•	Xへの画像自動添付投稿は初期リリース対象外
	•	オフライン利用は保証しない

⸻

11. MVP定義

11-1. MVPに含めるもの
	•	風景画像1枚アップロード
	•	ペット画像最大3枚アップロード
	•	背景除去
	•	ペット画像の移動
	•	サイズ変更
	•	透明度変更
	•	レイヤー順変更
	•	PNGダウンロード
	•	X投稿画面を開く機能

11-2. MVPに含めないもの
	•	アカウント機能
	•	クラウド保存
	•	高度なフィルタ
	•	豊富なテンプレート
	•	自動レイアウト
	•	作品URL共有

⸻

13. 開発を爆速にするための実装方針

採用方針
	•	Vue 3 + TypeScript + Vite
	•	状態管理は軽量で済ませる
	•	UIライブラリを最小限にする
	•	まずPC向けUIを完成
	•	保存機能なし
	•	認証なし
	•	DBなし
	•	APIなし

開発上の割り切り
	•	出力形式は最初はPNGのみ
	•	ペット上限は3枚固定
	•	風景は1枚固定
	•	作品の再編集保存なし
	•	自動SNS画像投稿なし
	•	まず見た目を整えすぎず、機能優先

15. リリース判定基準

以下を満たしたら公開可能とする。
	•	主要ブラウザで風景画像アップロードができる
	•	ペット画像最大3枚の背景除去ができる
	•	位置・サイズ・透明度調整ができる
	•	PNGダウンロードが成功する
	•	X投稿ボタンで投稿画面を開ける
	•	致命的なUI崩れがない
	•	画像がサーバーに送信されていない
